php中变量的传值

我们可以把一个变量的值传给另一个变量，在传值时有两种情况：值传递和引用传递。

值传递：把一个变量的值在内存中复制一份给另一个变量（类似于克隆）

引用传递：把另一个变量在内存中的地址给另一个变量，两个变量实际上操作同一块内存（类似于起别名）

值传递：

值传递直接使用 = 即可

示例代码：把a变量的值传给b

引用传递

引用传递时在变量名前使用 & 符号，获取变量在内存中的地址

示例代码：把a的引用传给b

说明：

1、如果不加 & 符号默认都是值传递

2、对象类型的数据除外，对象类型的数据都是引用传递

php中常量与变量

php中的数据分为两大类：常量和变量

常量：值不能被改变。比如：圆周率、光速、一周的天数、一天的小时数等等。

变量：值可以被改变。比如：年龄、价格、身高、车速、风速等等。

常量的定义和使用

可以用 const 或者 define 来定义一个变量

const 语法：const 名称=值;

define 语法：define(‘名称’,值);

命名规范：常量名都是用大写：比如：PI、Light_SPEED。

Apache配置

配置文件

Apache 运行时依赖于配置文件，在这个文件中定义了端口号、网站根目录、权限、虚拟主机等信息 。配置文件的的位置：

PHPStudy 安装目录\PHPTutorial\Apache\conf\http.conf

修改配置文件时要注意：

1、配置文件以 # 开头的行是被注释的行，不起作用。

2、每次修改了配置文件之后，需要重新启动Apache才会生效

3、常用配置说明：

配置项 说明
Listen 监听的端口号
LoadModule 加载其他功能模块，需要加载PHP模块才能运行PHP
DocumentRoot 配置网站根目录，只有这个目录下的网页别人才能看见
DirectoryIndex 当用户没用写请求的文件时默认访问的文件
Include 加载其他配置文件

配置端口号

Apache 默认监听80端口，如果希望监听其他端口可以修改listen这一项

配置网站根目录

当用户访问一个文件时，Apache 会到网站根目录中找到这个文件。比如：网站的根目录时 C:/www, 那么会有以下对应关系：

用户访问路径 服务器上对应的文件
http://localhost/user.php C:/www/user.php
http://localhost/goods/cart/goods.php C:/www/goods/cart/goods.php

在配置文件中我们可以通过 DocumentRoot 来修改网站的根目录

nodejs基础

1、NodeJs概述

1.1、NodeJs介绍

http://nodejs.cn

NodeJs是一个基于谷歌V8引擎的js的运行环境

运行环境即需要执行必须要存在的支撑。例如：Java语言的开发也是需要JRE支持。NodeJs基于js语法

1.2、NodeJs特点

①nodejs基于单线程方式工作

②采用事件驱动的方式。类似于浏览器端执行js过程中绑定的各种事件。一旦事件触发就执行对应的程序

③拥有全球最大开源生态系统（npm）

④非阻塞模式的I/O（输入于输出）模型。
